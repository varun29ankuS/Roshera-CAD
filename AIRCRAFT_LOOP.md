# Aircraft design loop — entangled state

## Mission
Design a complete, fully-verified aeroplane. **Aeroplane-capable** = every manifest
component builds `sound` AND the harness verifies every step AND zero open bugs.

## The entangled cycle (per iteration)
1. **DESIGN** the next unbuilt component (MCP build + blackboard justification + `verify_claim` grounding).
2. **HARNESS**: `ground_truth` EVERY step → `sound=true` required; no unsound part advances.
3. **BUGS**: any `sound=false` / op error → gated fundamental fix (worktree) → rebuild+restart → resume.

Each iteration tightens all three: design surfaces bugs + exercises the harness; the harness gates
design + grows new invariants where it can't verify; fixed bugs unblock + raise capability.
